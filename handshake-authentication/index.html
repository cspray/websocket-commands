Handshake Authentication

Handshake authentication takes care of making sure that the Websocket is authenticated properly during the handshake procedure. You can conceptually think of this as handling the traditional HTTP request before websocket communication begins. You can interrogate a Request object and set the status code, headers, and/or cookies for the Response.

HandshakeAuthenticator::onHandshake

This method must return a Promise that should resolve with the Response object. It accepts 2 arguments; the first being an Amp\Http\Server\Request which can be used to interrogate headers, cookies, or query parameters and the second being an Amp\Http\Server\Response which can be used to set status code, headers, and/or cookies. If the Response status code is set to a non-2XX response the websocket connection will be terminated.
